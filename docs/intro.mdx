# Introduction

Let's discover **O-Replay in less than 5 minutes**.

## Structure {#structure}

Races in O-Replay are organized into events and stages.

For example, if you are running a weekend championship with middle and long distance races,
you would create one event "Championship" with two stages:
"Middle Distance" and "Long Distance".

Each stage can represent a different orienteering discipline.

For a single race, you would create an event "Long Distance" with a single stage "Long Distance".

## Workflow Summary {#workflow-summary}

There are two main components in O-Replay:
1. **O-Replay WEB**: *Manage your events*
    - Log In to Create and Manage your events.
    - Add the different stages of your event.
    - Generate the `API Key` for this event.
    - Share a `QR-code` of this event.
2. **O-Replay Desktop Client**: *Upload Data (results, start lists...)*
    - Use the `API Key` from the O-Replay WEB.
    - Select the stage you want to upload data to.
    - Select the folder where such data is.
    - Start the Upload Process.
