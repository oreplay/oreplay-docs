# Introducción

Descubrimos **O-Replay en menos de 5 minutos**.

## Estructura

Las carreras en O-Replay se organizan en eventos y etapas.

Por ejemplo, si organizas un campeonato de fin de semana con carreras de media y larga distancia,
entonces crearías un evento "Campeonato" con dos etapas:
"Media Distancia" y "Larga Distancia".

Cada etapa puede representar una disciplina de orientación diferente.

Para una sola carrera, crearías un evento "Larga Distancia" con una sola etapa "Larga Distancia".

## Resumen del Flujo

Hay dos componentes principales en O-Replay:
1. **O-Replay WEB**: *Gestiona tus eventos*
    - Inicia Sesión para Crear y Gestionar tus eventos.
    - Añade las distintas etapas de tu evento.
    - Genera la `Clave Secreta` para este evento.
    - Comparte un `Código QR` de este evento.
2. **O-Replay Cliente de Escritorio**: *Sube Datos (resultados, horas de salida...)*
    - Use la `Clave Secreta` de O-Replay WEB.
    - Selecciona la etapa a la que quieras subir datos.
    - Selecciona la carpeta donde se almacenan estos datos.
    - Inicia el Proceso de Subida.
